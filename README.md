# Origin Star Map · 星火档案

一个 AI 原生的知识探索游戏：模型会为陌生领域实时编写可操作的 HTML / CSS / SVG / Canvas / WebGL 世界，让好奇心在星图中连成路径。

传统教育要求人先选择一个专业，再沿着它深入；星火档案尝试另一条路——先让玩家进入许多陌生世界，各获得一个能够继续提问的起点。AI 不只是回答问题，而是实时生成观测、判断反馈、知识解释和下一条跨领域航线。

## 核心玩法

- **驾驶**：用 WASD / 方向键操纵飞船，Shift 加速，Space 发射探针；穿过航线信标可以带走更多知识样本。
- **抵达**：航行与扫描本身持续可玩，AI 会在后台同时生成目的地，不再用加载页打断探索。
- **点**：完成一次交互式观测，获得一个知识入口。
- **边**：选择下一条航线，留下两个领域之间的真实联系。
- **面**：每三次观测闭合成一张解释面，AI 提炼它们共同解释的问题。
- **体**：每三张解释面生成一个世界模型，得到可以迁移到新问题的思考工具。

星图会长期保存玩家探索过的领域、做过的判断、选择的路径以及逐渐涌现的知识结构。

飞行不是等待动画。玩家选择直达、偏航采样或远距探测，会形成不同的航行记录；AI 会据此调整下一批航线的跨度与切入角度，让驾驶行为真正改变知识宇宙。

点、线、面、体只描述知识如何生长，并不限制一个知识点如何呈现。每次观测都可以由模型生成完全不同的浏览器原生交互程序：轨道实验、生态模拟、城市流量、声音波形、机械结构、CSS 3D、WebGL 空间或逐帧代码动画。生成代码在无网络权限的隔离沙箱中运行；固定控件只作为失败兜底。

游戏与模型之间使用开放场景协议，而不是一组固定的 SVG 模板。场景可以自由使用浏览器原生能力，并通过 `SparkRuntime` 回传进度、触觉信号和最终结果。未来更强的代码模型出现时，只需更换模型配置，不需要为每一种新表现形式重写产品界面。

## 本地运行

需要 Node.js `>=22.13.0` 和一个阿里云百炼 DashScope API Key。也可以连接任何提供 OpenAI-compatible Chat Completions 接口的模型服务。

```bash
npm install
Copy-Item .env.example .env
npm run dev
```

在 `.env` 中配置：

```env
DASHSCOPE_API_KEY=your_key_here
```

然后访问 `http://localhost:3000`。

密钥只由服务端 API 读取；`.env` 已加入 `.gitignore`，不要把真实 Key 提交到仓库。

## 技术栈

- React + TypeScript
- vinext / Vite
- DashScope Qwen
- 可替换的场景模型与知识模型
- 模型实时生成的 HTML、CSS、SVG、Canvas、WebGL、Web Audio 与代码动画世界
- 沙箱隔离、CSP 与受控消息桥
- LocalStorage 个人星图存档

## 产品理念

AI 可以回答几乎一切，但不能替人拥有问题。

当一个人对某个领域完全陌生时，他缺少的往往不只是答案，而是提出第一个好问题的能力。这个项目希望帮助玩家略知一二地走进一百个领域，在大脑里留下星星点点的知识萌芽，并让这些点逐渐连接成新的可能性。
